# Style & Branding Guide

Librariarr uses the **Lagoon** design system — a deep-ocean dark interface anchored to the product's own logo colors: the cyan reel arc and the gold spine. Surfaces share a blue hue whose chroma *rises* with elevation, so lifted panels read like warmer water above a dark floor. Chrome recedes (the sidebar is darker than the canvas); content lifts.

## Brand Identity

### Logo

The logo is an SVG side view of a movie reel with a circular arrow (representing lifecycle/recycling) enclosing book spines (representing a media library). The design system derives directly from it:

| Element | Color | Hex | System role |
|-|-|-|-|
| Circular arc / arrow | Cyan | `#00a4dc` | `--brand` (≈ `#09b7dc`) |
| Spine 1 (gold) | Gold | `#e5a00d` | `--amber` highlight family |
| Spine 2 (blue) | Blue | `#00a4dc` | — |
| Spine 3 (green) | Green | `#52b54b` | `--green` semantic family |
| Spine 4 (orange) | Orange | `#e4881c` | — |
| Spine 5 (purple) | Purple | `#9b59b6` | — |
| Background fill | Deep navy | `#0f1923` | Surface hue reference |

### Application Name

"Librariarr" is displayed in the sidebar using the display font (Space Grotesk, semibold, tight tracking). The "arr" suffix renders in `text-brand-bright`.

## Color System

All colors use the **OKLCH color model** for perceptual uniformity. The application enforces dark mode only (`className="dark"` on `<html>`). Derived tints use `color-mix(in oklch, …)` so they follow the active accent automatically.

### Surfaces

Surfaces share hue ≈ 235–236 with chroma rising alongside lightness — elevation reads as both brighter *and* richer:

| Token | OKLCH Value | Hex | Role |
|-|-|-|-|
| `--sidebar` | `oklch(0.13 0.015 235)` | `#03080c` | Navigation rail — **below** the canvas |
| `--background` / `--surface-0` | `oklch(0.16 0.018 235)` | `#060f14` | Page canvas |
| `--card` / `--surface-1` | `oklch(0.20 0.02 236)` | `#0d181e` | Cards and panels |

| `--muted` | `oklch(0.23 0.02 236)` | — | Muted backgrounds |
| `--surface-2` | `oklch(0.24 0.022 236)` | `#152128` | Hover fills, wells |
| `--accent` | `oklch(0.26 0.024 236)` | — | shadcn hover fill |
| `--secondary` / `--surface-3` | `oklch(0.28 0.024 236)` | `#1d2b33` | Highest resting surface |

Each has a solid and a `-dim` (≈15% alpha) fill variant:

| Token | OKLCH Value | Role |

### Borders

Hairlines carry a faint cyan tint: `--border` = `oklch(0.85 0.06 220 / 10%)`, `--input` @ 14%, `--border-strong` @ 16%.

Presets override the **full brand family** (`--brand`, `--brand-bright`, `--brand-dim`, `--brand-faint`, `--on-brand`, `--primary`, `--primary-foreground`, `--ring`, `--sidebar-primary`, `--sidebar-primary-foreground`, `--chart-1`), so the sidebar, mobile tab bar, atmosphere, and charts all follow:

| Preset | Base OKLCH | Swatch |
|-|-|-|
| Lagoon (default) | `oklch(0.72 0.13 220)` | `#09b7dc` |
| Blue | `oklch(0.66 0.17 255)` | `#3f93f7` |
| Violet | `oklch(0.68 0.20 300)` | `#ad74ff` |
| Green | `oklch(0.76 0.15 165)` | `#2cce99` |
| Orange | `oklch(0.75 0.16 60)` | `#f6922e` |
| Rose | `oklch(0.70 0.19 10)` | `#fc6183` |
| Teal | `oklch(0.74 0.13 180)` | `#1dc5ae` |

All accents sit at L ≥ 0.66 so dark-on-accent text stays legible. Preset *names* are persisted in `AppSettings.accentColor` — never rename them.

## Typography

Three font families from Google Fonts (loaded via `next/font`):

### Space Grotesk (Display)

**Usage:** Page headings, sidebar brand name, section titles.
**Weights:** 400, 500, 600, 700
**Tailwind class:** `font-display`

A grotesque with distinctive ink traps and character — applied to all `<h1>` elements with `font-display tracking-tight`.

### Geist (Body)

**Usage:** All body text, navigation items, labels, inputs, buttons.
**Tailwind class:** `font-sans` (default — applied on `<body>` in globals.css)

Vercel's neutral geometric sans; crisp at UI sizes.

<Aside type="caution">
`font-sans` must stay applied on `body` in `globals.css`. The `next/font` CSS variables are defined on `<body>`, so Tailwind preflight's `html` font rule cannot see them — without the body rule the UI silently falls back to the system font stack.
</Aside>

### Geist Mono (Monospace)

**Usage:** System log console, eyebrows, technical data, `kbd` hints.
**Weights:** 400, 500, 600
**Tailwind class:** `font-mono`

The **mono-for-data** pattern anchors the aesthetic: timestamps, counts, file sizes, and uppercase eyebrow labels (`.eyebrow`) always render in mono.

### Type Scale

| Context | Classes |
|-|-|
| Page title | `text-2xl sm:text-3xl font-bold font-display tracking-tight` |
| Card title | `text-lg font-semibold` |
| Section heading | `text-xl font-bold font-display` |
| Body text | `text-sm` (14px) — default |
| Helper/metadata | `text-xs text-muted-foreground` |
| Eyebrow | `.eyebrow` (mono, 10.5px, uppercase, 0.18em tracking) |
| Micro text | `text-[10px]` or `text-[11px]` |

## Effects & Atmosphere

### Glass Morphism

`.glass` — translucent card surface for chrome that floats over content (filter bars, mobile header, **bottom tab bar**, login card). Derived from `--card` via `color-mix`, with `backdrop-filter: blur(20px)` on desktop and a near-opaque fallback on mobile (blur is disabled < 768px to prevent scroll lag).

### Canvas Atmosphere

`.canvas-atmosphere` — a faint brand-tinted radial wash at the top of the main content area (7% brand). Because it uses `color-mix(… var(--brand) …)`, it follows the user's accent.

### Auth Backdrop

`.auth-backdrop` — full-screen gradient behind login/onboarding cards: an 18% brand bloom at the horizon over a deep vertical falloff.

### Focus Ring

Focus-visible states use a cyan glow instead of a hard outline:

```css
box-shadow: 0 0 0 2px var(--ring),
            0 0 14px color-mix(in oklch, var(--brand) 30%, transparent);
```

### Selection

Text selection uses the brand color at 35% opacity.

### Custom Scrollbar

All scrollable areas use thin, translucent scrollbars (`scrollbar-width: thin`).

### Shadows

| Token | Usage |
|-|-|
| `--shadow-card` | Resting cards — inset top light edge + soft ambient drop |
| `--shadow-pop` | Popovers, dialogs, hover-lifted media cards |

## Animation

### Entry Animations

Page elements use a staggered fade-in-up animation on load (350ms ease-out). Apply `animate-fade-in-up` for single elements, or `stagger-children` on a parent to auto-stagger up to 12 children (40ms intervals). Both are disabled under `prefers-reduced-motion`.

### Transitions

| Context | Duration | Easing |
|-|-|-|
| Hover effects | 300ms | ease-out |
| Button press | 200ms | default |
| Sidebar collapse | 300ms | ease-in-out |
| Collapsible expand | 200ms | ease-out |
| Tab changes | instant | — |

### Micro-Interactions

- **Buttons:** `active:scale-[0.98]` press feedback
- **Media cards:** `hover:scale-[1.03]` with image `group-hover:scale-105`
- **Nav items:** `hover:translate-x-0.5` subtle rightward shift

## Mobile Web App

Librariarr is installable as a PWA (`manifest.ts`, `display: standalone`, theme color `#060f14`) and the layout adapts below the `md` (768px) breakpoint:

### Bottom Tab Bar

`MobileTabBar` (`src/components/mobile-tab-bar.tsx`) — fixed glass bar with five primary destinations: Home, Library, Lifecycle, Tools, Settings. The active tab renders in `text-brand-bright` with a `bg-brand-dim` pill behind the icon. The full navigation (sub-pages, Logs, Query) remains in the drawer behind the header menu button.

### Safe Areas

The viewport uses `viewport-fit=cover`; utilities pad content clear of notches and the home indicator (`env()` resolves to 0 in regular browser tabs):

| Class | Padding |
|-|-|
| `.pt-safe` | `env(safe-area-inset-top)` — mobile header, drawer |
| `.pb-safe` | `env(safe-area-inset-bottom)` — tab bar |
| `.pb-tabbar` | tab bar height + bottom inset on the main scroll container (0 at `md+`) |

The tab bar height is `--tabbar-height` (4rem). Floating elements (BackToTop, toasts via Sonner's `mobileOffset`) offset above it.

### Touch

Tab bar targets are full-height grid cells (≥ 64px). The iOS status bar style is `black-translucent`; the mobile header and drawer pad with `.pt-safe` so content clears the notch.

## Component Patterns

### Cards

Default card styling: `bg-card rounded-xl border shadow-[var(--shadow-card)]` — an inset top highlight simulates a light edge for perceived depth.

### Buttons

Six variants: `default`, `destructive`, `outline`, `secondary`, `ghost`, `link`
Eight sizes: `default` (h-9), `xs` (h-6), `sm` (h-8), `lg` (h-10), `icon` (9), `icon-xs` (6), `icon-sm` (8), `icon-lg` (10)

All buttons include `active:scale-[0.98]` and `transition-all duration-200`. Default buttons are brand cyan with dark (`--on-brand`) text.

### Badges

Rounded-full pills with `backdrop-blur-sm` for use over images. Variants: `default`, `secondary`, `destructive`, `outline`, `ghost`, `link`.

### Media Cards

Poster-forward cards with cinematic hover:

```
hover:scale-[1.03]
hover:shadow-[var(--shadow-pop)]
hover:ring-1 hover:ring-white/10
transition-all duration-300 ease-out
```

### Tab Navigation

Border-bottom style tabs with icon support:

- **Active:** `border-primary text-foreground` with 2px bottom border
- **Inactive:** `border-transparent text-muted-foreground` with hover fade-in

### Sidebar Navigation

The rail sits **below** the canvas (`--sidebar` is darker than `--background`):

- **Active item:** `bg-brand-dim text-brand-bright` rounded fill
- **Inactive item:** muted, hover to `bg-surface-2 text-foreground`
- **Group labels:** `.eyebrow`-style mono uppercase
- **Search affordance:** bordered well with `⌘K` kbd hint, routes to the Query workspace

### Dialogs

Dialogs use `backdrop-blur-sm` on the overlay; destructive confirmations use `AlertDialog` with `variant="destructive"` actions.

### Hover Popovers

Every `MediaHoverPopover` must pass the **same universal set of fields** regardless of page, view type (table or card), or data source. All fields are optional — sections hide gracefully when data is absent. The component renders only what is non-null, so passing all fields is always safe.

**Universal field set:**

`title`, `year`, `summary`, `contentRating`, `rating`, `audienceRating`, `ratingImage`, `audienceRatingImage`, `duration`, `resolution`, `dynamicRange`, `audioProfile`, `fileSize`, `genres`, `studio`, `playCount`, `lastPlayedAt`, `addedAt`, `servers`, `seasonCount`, `episodeCount`, `albumCount`, `trackCount`, `qualityCounts`, `audioCodecCounts`

For grouped data sources (series shows, seasons, music artists, albums), metadata fields (`summary`, `genres`, `studio`, `contentRating`, `rating`, etc.) are aggregated from a representative child item via first-non-null picking in the API.

**Rules:**

- **Table and card views within the same page must always pass identical fields** — no exceptions
- `imageAspect="square"` for music items; `"poster"` (default) for everything else
- `audioCodecCounts` is music-only; `qualityCounts` is series-only
- When an item can be either individual or grouped (e.g. query results), conditionally pass the appropriate fields based on whether it's grouped (e.g. `seasonCount`/`episodeCount` for grouped series, `duration`/`resolution` for individual items)

## Page Layout Patterns

### Authenticated Pages

```
Desktop:  Sidebar (collapsible 72px/248px) | Main content (flex-1, overflow-y-auto)
Mobile:   Glass header (menu + logo) / Main content / Bottom tab bar
```

The main content area carries `.canvas-atmosphere` and `.pb-tabbar`.

### Page Headers

Every page follows this heading pattern:

```html
<h1 class="text-2xl sm:text-3xl font-bold font-display tracking-tight">
  Page Title
</h1>
<p class="text-muted-foreground mt-1">
  Brief description of the page purpose.
</p>
```

### Lifecycle Pages

All lifecycle pages (Rules, Matches, Pending Actions, Exceptions) include a `TabNav` for filtering by media type (Movies / Series / Music), plus an "All" tab on Matches and Pending Actions.

### Login / Onboarding

Full-screen `.auth-backdrop` with a glass-morphism card and the pulsing `animate-logo-glow` brand glow on the logo.

## Don'ts

- **Don't use flat grey** — all dark surfaces carry the blue tint (hue ~235)
- **Don't use HSL** — the entire color system is OKLCH for perceptual consistency
- **Don't hardcode brand hues** — derive tints with `color-mix(in oklch, var(--brand) …)` so accents keep working
- **Don't use generic fonts** — never fall back to Inter, Roboto, Arial, or system fonts
- **Don't edit shadcn/ui primitives** in `src/components/ui/` unless adding project-wide enhancements
- **Don't use light mode** — the application enforces dark mode only
- **Don't rename accent preset names** — they're persisted in user settings
- **Don't add emojis** — use Lucide icons for all iconography
- **Don't block the bottom tab bar** — floating mobile elements must offset by `--tabbar-height` + safe-area inset
